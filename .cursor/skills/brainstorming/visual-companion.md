# Visual Companion Guide

Use this companion only when a visual artifact will improve understanding more than text.

## Offer Gate

Send this offer as a standalone message, with no other content:

> Some of what we're working on might be easier to explain if I can show it to you in a web browser. I can put together mockups, diagrams, comparisons, and other visuals as we go. This feature is still new and can be token-intensive. Want to try it? (Requires opening a local URL)

Wait for consent before using browser visuals.

## Per-Question Decision Rule

Ask: "Would the user understand this better by seeing it than reading it?"

- Use visual companion for mockups, wireframes, layouts, flows, diagrams, and side-by-side visual comparisons.
- Use text-only chat for requirements, constraints, priorities, trade-offs, and conceptual decisions.

## Working Style

- Keep each visual focused on one question.
- Label options clearly (A/B/C) and keep wording neutral.
- After showing visuals, ask one question and wait for response.
- Prefer fast, low-fidelity visuals early; increase fidelity only after direction is clear.

## Safety and Scope

- Do not use visuals to bypass the brainstorming process.
- Do not implement or scaffold code while in design phase.
- Keep visuals aligned with confirmed constraints and project context.
