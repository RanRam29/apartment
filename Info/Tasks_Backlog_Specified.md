# חלק ג — איכות, עלות ובטיחות של Gemini

## משימה-AI-020 — Rate limiting ייעודי לקריאות Gemini

| שדה | תוכן |
|-----|------|
| **מזהה** | AI-020 |
| **עדיפות** | P1 |
| **מטרה** | להגן מפני ניצול, חריגת תקציב API ו-DoS לוגי על נקודות הקצה שקוראות ל-Gemini. |
| **היקף** | מגבלות נפרדות (או משוקללות) ל-`POST /api/recommendations/search` ול-`POST /api/apartments/:id/marketing-copy`; זיהוי לפי משתמש מחובר + IP כגיבוי. |
| **מחוץ להיקף** | חיוב לפי שימוש בתוך האפליקציה (זה מוצר נפרד). |

| **קריטריוני קבלה** | בדיקות אוטומטיות או ידניות: חריגה ממכסה מחזירה 429; קריאות לגיטימיות עוברות. |
| **תלויות** | אין חובה; ניתן לבנות על `express-rate-limit` הקיים ב-[`backend/src/app.js`](../backend/src/app.js). |
| **הערות טכניות** | להפריד מ-limit גלובלי של 200 דקה כדי לא לחנוק את כל ה-API. |
