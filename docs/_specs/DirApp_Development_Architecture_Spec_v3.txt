# 4. Services Architecture
## 4.1 kycService
initiateVerification(userId) — שולח ל-Persona, מחזיר verification URL
handleWebhook(payload, hmacSignature) — HMAC-SHA256 validation + עדכון DB
checkAndUnlockContracts(userId) — לאחר APPROVED — מאפשר חתימה
abstraction layer — IKYCProvider interface להחלפת Persona בעתיד
## 4.2 contractService
uploadAndExtract(file, landlordId) — Gemini OCR + שמירה ב-R2
validateGate(agreementId) — בודק כל תנאי Validation Gate
transitionState(agreementId, newState) — StateLockGuard
generateLedger(agreementId) — יצירת כל שורות הלדגר
scheduleExpiringAlerts(agreementId) — יצירת כל ההתראות
## 4.3 notificationService
sendPush(userId, title, body, data) — Expo Push
sendEmail(to, template, data) — Resend
scheduleReminder(userId, timestamp, payload) — תזמון עתידי
## 4.4 geminiService (קיים — מורחב)
extractContractFields(fileBuffer) — חדש: OCR + structured extraction
generateMarketingCopy(apartmentData) — קיים
parseNLPSearch(query) — קיים
scoreCompatibility(tenant, apartment) — קיים
# 5. Security Architecture
## 5.1 Authentication
JWT tokens — חתימה + expiry
Role-based middleware — landlord / tenant / admin
KYC guard — routes מסוימות דורשות KYC APPROVED
## 5.2 KYC Security
HMAC-SHA256 על כל Persona webhook
ספרת ביקורת ת"ז — pre-filter לפני שליחה ל-Persona
S3 presigned URLs — 5 דקות בלבד לעלות מסמכים
תמונות KYC נמחקות אוטומטית אחרי 7 ימים
## 5.3 Data Privacy
תיקון 13 לחוק הגנת הפרטיות — data minimization
R2 lifecycle rules — מחיקה אוטומטית לפי סוג bucket
לא שומרים מסמכי זהות גולמיים — רק Persona inquiry ID
# 6. Cloudflare R2 — Storage Strategy
דחיסה: אוטומטית בצד הלקוח לפני העלאה — מקסימום 2MB לתמונה.
